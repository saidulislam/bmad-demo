# The BMAD Method

**BMAD** stands for **Breakthrough Method for Agile AI-Driven Development**.

BMAD is a free, open-source framework created by **Adam Blackington** that brings structure, repeatability, and accountability to AI-assisted software development. Rather than treating an LLM as a single generalist chatbot, BMAD organizes AI collaboration around **specialized agent roles** that mirror a real agile development team.

The core philosophy is **documentation-first development**: specifications (PRDs, architecture documents, user stories) are the source of truth, and code becomes a downstream derivative. This is a deliberate counter to "vibe coding" -- the ad-hoc, conversational style of prompting AI to write code without planning.

---

## The Agent System

BMAD defines 9+ specialized AI agents, each represented as an "Agent-as-Code" Markdown file describing expertise, responsibilities, constraints, and expected outputs:

| Agent | Role |
|-------|------|
| **Orchestrator** | Coordinates the overall workflow |
| **Analyst** | Requirements gathering and research |
| **Product Manager** | Writes PRD and acceptance criteria |
| **Architect** | System and technical design |
| **UX Designer** | User flows and UI specifications |
| **Scrum Master** | Prioritizes work, generates sprint stories |
| **Developer** | Implements features incrementally |
| **QA Engineer** | Testing and validation |
| **Tech Writer** | Documentation |

Each agent has a tightly scoped context, receives only the artifacts it needs, and produces well-defined outputs.

---

## Four-Phase Workflow

1. **Analysis** -- Creates a concise PRD capturing the problem statement and constraints.
2. **Planning** -- Breaks the PRD into user stories with explicit acceptance criteria, prioritized by the Scrum Master persona.
3. **Solutioning** -- The Architect drafts minimal design; the Developer outlines implementation steps.
4. **Implementation** -- Iterative execution using small stories with clear criteria. Artifacts update incrementally rather than being rebuilt.

Each phase transition requires **quality gates** -- verification that artifacts meet defined criteria before handoff to the next agent. This prevents inconsistencies, missing requirements, and coverage gaps from propagating downstream.

---

## Three Complexity Tracks

- **Quick Flow** -- For bug fixes and small changes; technical spec only, under 5 minutes.
- **Standard** -- Full PRD, architecture, and UX for products and MVPs.
- **Enterprise** -- Extended documentation, audit trails, and compliance artifacts.

---

## Key Principles

- **Documentation is the source of truth**, not chat history or code alone.
- **Scoped context per agent** -- each agent sees only what it needs, maximizing context window efficiency.
- **Incremental verification** -- small stories enable easier review and rollback.
- **Persistent artifacts** -- agent handoffs and versioned documents maintain coherence across the entire project lifecycle, solving the "AI forgets between sessions" problem.
- **Lightweight adoption** -- uses Markdown conventions and integrates with existing tools.

BMAD connects AI agents to real development stacks through the Model Context Protocol, enabling integration with SonarQube, Atlassian (Jira/Confluence), GitHub, and Figma.

---

## Installation

Requires Node.js v20+:

```bash
npx bmad-method install
```

After installation, invoke `bmad-help` inside your AI IDE and it will inspect your project and show available options.

---

## Why It Matters

BMAD solves three fundamental problems with AI coding:

1. **Unpredictability** -- structured agents produce consistent outputs
2. **Hallucination/drift** -- quality gates catch issues between phases
3. **Context loss** -- persistent artifacts maintain coherence across sessions
