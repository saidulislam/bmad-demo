## Installation Summary

```bash
npx bmad-method install \
  --directory /path/to/your/project \
  --modules "bmm,tea" \
  --tools "claude-code" \
  --user-name "Saidul" \
  --yes
```

| Flag | Purpose |
|------|---------|
| `--modules "bmm,tea"` | Install BMad Method (agile SDLC) + Test Architect (quality gates) |
| `--tools "claude-code"` | Integrate with Claude Code IDE (53 slash commands) |
| `--user-name "Saidul"` | Name agents use to greet and address you in conversations |
| `--yes` | Accept all defaults without interactive prompts — repeatable and scriptable |

### What Was Installed

| Component | Details |
|-----------|---------|
| **Core** | BMAD core platform with help, brainstorming, editorial tools |
| **BMM** (BMad Method) | Full agile SDLC framework — 34+ workflows, 9 agent personas |

| **Claude Code Integration** | 53 skills available as slash commands |

## Workflow Phases & Skills

### Getting Started

| Skill | Command | Description |
|-------|---------|-------------|
| **Help** | `/bmad-help` | Analyzes current state and recommends next skill(s) to use |
| **Init** | `/bmad-init` | Initialize BMAD project configuration and load config variables |

### Phase 1 — Analysis

| Skill | Command | Description |
|-------|---------|-------------|
| **Product Brief** | `/bmad-product-brief` | Create or update product briefs through guided or autonomous discovery |
| **Domain Research** | `/bmad-domain-research` | Conduct domain and industry research |
| **Market Research** | `/bmad-market-research` | Conduct market research on competition and customers |
| **Technical Research** | `/bmad-technical-research` | Conduct technical research on technologies and architecture |
| **Document Project** | `/bmad-document-project` | Document brownfield (existing) projects for AI context |

### Phase 2 — Planning

| Skill | Command | Description |
|-------|---------|-------------|
| **Create PRD** | `/bmad-create-prd` | Create a Product Requirements Document from scratch |
| **Edit PRD** | `/bmad-edit-prd` | Edit an existing PRD |
| **Validate PRD** | `/bmad-validate-prd` | Validate a PRD against standards |
| **Create UX Design** | `/bmad-create-ux-design` | Plan UX patterns and design specifications |

### Phase 3 — Solutioning

| Skill | Command | Description |
|-------|---------|-------------|
| **Create Architecture** | `/bmad-create-architecture` | Create architecture and solution design decisions |
| **Create Epics & Stories** | `/bmad-create-epics-and-stories` | Break requirements into epics and user stories |
| **Check Implementation Readiness** | `/bmad-check-implementation-readiness` | Validate PRD, UX, Architecture and Epics specs are complete |
| **Generate Project Context** | `/bmad-generate-project-context` | Create project-context.md with AI rules |

### Phase 4 — Implementation

| Skill | Command | Description |
|-------|---------|-------------|
| **Sprint Planning** | `/bmad-sprint-planning` | Generate sprint status tracking from epics |
| **Create Story** | `/bmad-create-story` | Create a dedicated story file with full implementation context |
| **Dev Story** | `/bmad-dev-story` | Execute story implementation following the story spec file |
| **Quick Dev** | `/bmad-quick-dev` | Implement any intent — build, fix, tweak, refactor, add, or modify code |
| **Sprint Status** | `/bmad-sprint-status` | Summarize sprint status and surface risks |
| **Code Review** | `/bmad-code-review` | Adversarial review using parallel review layers (Blind Hunter, Edge Case Hunter, Acceptance Auditor) |
| **Correct Course** | `/bmad-correct-course` | Manage significant changes during sprint execution |
| **Retrospective** | `/bmad-retrospective` | Post-epic review to extract lessons and assess success |

## Recommended Workflow for a New Project

This is the end-to-end path from idea to working software:

```
Step 1: /bmad-help                          ← Get oriented
Step 2: /bmad-product-brief                 ← Define the product idea (Analyst: Mary)
Step 3: /bmad-create-prd                    ← Full requirements doc (PM: John)
Step 4: /bmad-validate-prd                  ← Quality gate check
Step 5: /bmad-create-ux-design              ← UX specifications (UX: Sally)
Step 6: /bmad-create-architecture           ← Technical design (Architect: Winston)
Step 7: /bmad-create-epics-and-stories      ← Break into epics & stories (SM: Bob)
Step 8: /bmad-check-implementation-readiness ← Quality gate — are we ready to build?
Step 9: /bmad-sprint-planning               ← Plan the first sprint
Step 10: /bmad-create-story                 ← Create detailed story spec
Step 11: /bmad-dev-story                    ← Implement the story (Dev: Amelia)
Step 12: /bmad-code-review                  ← Review the code
Step 13: /bmad-sprint-status                ← Check progress
        ... repeat Steps 10-13 per story ...
Step 14: /bmad-retrospective                ← Lessons learned after epic completion
```

### Quick Flow (Bug Fixes / Small Changes)

For small tasks that don't need the full ceremony:

```
/bmad-agent-quick-flow-solo-dev    ← Talk to Barry for rapid spec + implementation
   or
/bmad-quick-dev                    ← Jump straight to code changes
```

## Team Rollout Recommendations

### For Engineering Teams at Enterprise Scale

1. **Start with a pilot project** — Run one team through the full workflow on a real but low-risk project
2. **Install BMM + TEA** — TEA is critical for enterprise quality standards and traceability
3. **Use Claude Code** as the primary IDE integration
4. **Standardize installation** with the non-interactive command:
   ```bash
   npx bmad-method install \
     --directory /path/to/project \
     --modules "bmm,tea" \
     --tools "claude-code" \
     --user-name "Team Name" \
     --yes
   ```
5. **Enforce quality gates** — Phase transitions (Analysis → Planning → Solutioning → Implementation) should require `/bmad-check-implementation-readiness` before coding begins
6. **Leverage TEA for compliance** — Use `/bmad-testarch-trace` for traceability matrices and `/bmad-testarch-ci` for CI/CD quality pipeline scaffolding
7. **Track progress** — Use `/bmad-sprint-status` for visibility across sprints

### Key Benefits for Enterprise Adoption

- **Consistency** — Every project follows the same structured workflow
- **Auditability** — Documentation-first approach creates a paper trail (PRD → Architecture → Stories → Code)
- **Quality** — Built-in quality gates prevent incomplete specs from reaching implementation
- **Onboarding** — New team members follow the same process; agents guide them
- **Context preservation** — Artifacts persist across sessions, solving the "AI forgets" problem
